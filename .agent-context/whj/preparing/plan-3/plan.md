# 网站分组（文件夹）

> 状态: 未执行

## 目标

实现类似手机桌面的分组文件夹功能。支持 1×2、2×1、2×2 三种尺寸，文件夹内直接展示缩小版网站图标，点击即可打开网站，无需展开文件夹。支持通过拖拽和批量操作将网站移入分组。完成后可创建不同尺寸的分组，将网站拖入/拖出分组，直接在标签页上交互。

## 内容

### 1. 分组数据模型与状态管理

- 定义 `Group` 类型：`id`、`name`、`size`（`1x2` | `2x1` | `2x2`）、`order`、`children: string[]`（网站 ID 列表）
- 更新 IndexedDB schema：添加 `groups` 表
- 创建 `groupsStore`（Preact Signals）：分组列表 signal
- 实现 CRUD：`addGroup`、`updateGroup`、`removeGroup`
- 实现成员管理：`addToGroup`、`removeFromGroup`、`reorderInGroup`
- **批量加入**：`batchAddToGroup(groupId, websiteIds[])` 方法，供批量操作调用
- 完成标准：store 方法可独立调用，分组数据持久化正确

### 2. 分组尺寸与内部布局规则

- **1×2**（宽 1 列 × 高 2 行）：内部 1 列 × 3 行，显示最多 3 个缩小图标
- **2×1**（宽 2 列 × 高 1 行）：内部 3 列 × 1 行，显示最多 3 个缩小图标
- **2×2**（宽 2 列 × 高 2 行）：内部 3 列 × 3 行，显示最多 9 个缩小图标
- 超出容量的图标不展示，右下角显示 `+N` 溢出计数
- 完成标准：三种尺寸渲染符合规则，溢出计数准确

### 3. 分组容器组件

- 遵循 `ui-ux-pro-max` 设计系统
- 外壳：圆角矩形、半透明/毛玻璃背景、底部分组名称标签
- 内部：CSS Grid 子网格渲染缩小版图标
- 缩小版图标组件：保持与标准卡片一致的风格，但缩小适配分组内尺寸；同样支持自定义图标（base64）
- 在 CSS Grid 中通过 `grid-column: span N` / `grid-row: span N` 占据对应区域
- 完成标准：分组在主网格中正确占位，内部图标渲染清晰

### 4. 分组内交互

- 点击分组内小图标 → 直接打开对应网站（无需展开分组）
- 图标 hover 状态反馈（高亮/缩放）
- 分组整体 hover 时轻微提升效果
- 完成标准：分组内图标可直接点击打开网站，交互流畅

### 5. 分组管理 UI

- 创建分组：长按/右键网格空白区域或通过菜单触发
- 创建表单：输入名称、选择尺寸（1×2 / 2×1 / 2×2 可视化选择）
- 编辑分组：右键分组 → 编辑（修改名称、调整尺寸）
- 删除分组：右键分组 → 删除，内部网站释放回主网格
- **接收批量移动**：plan-2 的批量操作「移动到分组...」需要展示可选分组列表；选择后调用 `batchAddToGroup`；若分组容量不足则提示并只移入可容纳的数量
- 完成标准：可通过 UI 完成分组的完整生命周期管理，包括接收批量移动的网站

### 6. 拖拽交互

- 拖拽网站到分组上方 → 网站加入分组（分组接收时高亮边框）
- 拖拽网站出分组到空白区域 → 网站脱离分组回到主网格
- 拖拽分组本身 → 分组在网格中重排（与普通网站混排）
- 拖拽时碰撞检测：分组占据多格，需检测分组完整区域
- 完成标准：拖拽进出分组操作流畅，视觉反馈清晰

### 7. 网格集成

- 统一排序模型：网站和分组共用 `order` 序列
- 分组在 CSS Grid 中跨行/跨列，与 1×1 网站卡片混排
- 碰撞避免：放置分组时自动检测是否有空间，无空间时推挤后续项目
- 完成标准：分组与网站在同一网格中自然混排，无重叠
