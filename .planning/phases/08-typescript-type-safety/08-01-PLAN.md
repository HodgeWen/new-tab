  <files>types/ui.ts, store/grid-items.ts</files>
  <action>
In `types/ui.ts`, add two type guard functions after the existing type definitions:

```ts
export function isSiteItem(item: GridItemUI): item is SiteItemUI {
  return item.type === 'site'
}

export function isFolderItem(item: GridItemUI): item is FolderItemUI {
  return item.type === 'folder'
}
```

Then refactor `store/grid-items.ts` to use these type guards, replacing ALL `as SiteItemUI`, `as FolderItemUI`, `as GridItemUI` assertions. Specific replacements:

1. `syncList()` — replace `(item as FolderItemUI).sites = []` with `if (isFolderItem(item)) item.sites = []`
2. `syncList()` — replace `(item as SiteItemUI).pid` checks with `if (isSiteItem(item) && item.pid)` pattern
3. `syncList()` — replace `(folder as FolderItemUI).sites!.push(item as SiteItemUI)` with narrowed access via `isFolderItem(folder)` guard
4. `syncList()` filter — replace `!(item as SiteItemUI).pid` with type guard check
5. `toRecord()` — replace `const { sites: _, ...data } = item as FolderItemUI` with conditional destructure via `isFolderItem` check; for SiteItemUI just spread directly
6. `loadGridItems()` — replace `rest as GridItemUI`; the DB record type minus timestamps IS structurally GridItemUI, so use explicit type annotation on the destructured rest
7. `updateGridItem()` — same pattern as toRecord for sites stripping
8. `batchDeleteGridItems()` — replace `(child as SiteItemUI).pid === id` with `isSiteItem(child) && child.pid === id`

Import `isSiteItem, isFolderItem` from `@/types/ui` in grid-items.ts (alongside existing imports).

Do NOT add `as` assertions. Use discriminated union narrowing (type guards or direct `item.type === 'site'` checks) for every case.
